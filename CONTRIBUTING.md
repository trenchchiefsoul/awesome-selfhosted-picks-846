# Contributing

Thanks for wanting to add a project. Keep it simple — submit a PR editing `README.md`.

## Criteria

A project must be:

1. **Open-source** — with a recognized OSI license (MIT, Apache-2.0, AGPL, GPL, etc.).
2. **Self-hostable** or runnable locally — no required hosted-only tiers for core features.
3. **Actively maintained** — a commit within the last 6 months, or a clear "stable" status from maintainers.
4. **Useful to a general audience** — not a niche fork, demo repo, or wrapper.

## Format

Add the project alphabetically inside the most relevant category. Use this format:

```markdown
- [Project Name](https://github.com/owner/repo) — One-line description ending with a period.
```

- Keep descriptions under ~80 characters.
- Lead with the function, not marketing language ("Self-hosted X" beats "The best way to X").
- Don't invent new categories without discussion — open an issue first.

## Process

1. Fork the repo.
2. Edit `README.md`.
3. Open a PR with a short justification (link to repo, why it fits).
4. One project per PR.

Removals (unmaintained projects, broken links, license changes) are welcome too — same flow.
